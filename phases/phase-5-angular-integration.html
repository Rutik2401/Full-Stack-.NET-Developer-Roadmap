1 Standalone Components & Bootstrap

Angular 15+ introduced standalone APIs — components, directives, pipes that don't need a NgModule. Modern Angular apps are built standalone-first.

    provideRouter(routes),

🎯 Interview Points

  • Standalone removes need for declaring in NgModule
  • imports array on component now includes other components/directives
  • Lazy loading routes use loadComponent instead of loadChildren
  • Smaller bundles + simpler tree-shaking

2 Signals (Angular 16+) ⭐

Signals are Angular's new fine-grained reactive primitive. They replace much of RxJS for component state and enable a future without zone.js.

import { signal, computed, effect } from '@angular/core';

export class CartComponent {
  items = signal<CartItem[]>([]);
  total = computed(() => this.items().reduce((s, i) => s + i.price * i.qty, 0));
  count = computed(() => this.items().length);

  constructor() {
    effect(() => console.log('Cart updated:', this.total()));
  }

  add(item: CartItem) {
    this.items.update(arr => [...arr, item]);
  }

  setItems(arr: CartItem[]) { this.items.set(arr); }
}
<!-- template -->
<p>Items: {{ count() }}</p>
<p>Total: ₹{{ total() }}</p>

🎯 Signal vs Observable

SignalObservable
EagerYes — value always availableNo — must subscribe
SyncYesAsync
ForComponent state, derived valuesHTTP, events, time
SubscribeAuto via templateManual / async pipe

Signals are the future of Angular reactivity. Use signals for state; keep RxJS for streams (HTTP, WebSockets, events).

3 Change Detection & OnPush Strategy

By default Angular runs change detection on every async event in the app. OnPush limits a component to re-render only when its inputs change (by reference). Critical for performance.

@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  ...
})
export class ProductCard {
  @Input() product!: Product;   // re-renders only when reference changes

  constructor(private cdr: ChangeDetectorRef) {}

  forceUpdate() {
    this.cdr.markForCheck();   // schedule check
  }
}

🎯 OnPush Triggers Re-render When

  • An @Input() reference changes (immutability matters!)
  • An event handler in this component fires
  • An async pipe emits a new value
  • Manual cdr.markForCheck() or cdr.detectChanges()
  • Signals read in template change
⚠️ Gotcha: Mutating an array in place (arr.push(x)) won't trigger OnPush. Use immutable updates: arr = [...arr, x].

4 RxJS Mastery — switchMap & friends ⭐

Even working Angular devs are weak on higher-order operators. These show up in interviews and bug reports.

🎯 Higher-Order Mapping Operators (most asked)

OperatorBehaviorUse For
switchMapCancel previous inner if new arrivesSearch-as-you-type, latest only
mergeMap (flatMap)Run all in parallel, no orderIndependent parallel calls
concatMapQueue, run one at a time, in orderSequential writes, save order
exhaustMapIgnore new while one is runningLogin button (prevent double click)
 
// search box — cancels stale requests
search = new FormControl('');
results$ = this.search.valueChanges.pipe(
  debounceTime(300),
  distinctUntilChanged(),
  switchMap(term => this.api.search(term))
);

// parallel calls
combined$ = combineLatest([
  this.api.getUser(id),
  this.api.getOrders(id),
  this.api.getProfile(id)
]).pipe(map(([u, o, p]) => ({ user: u, orders: o, profile: p })));

// race / first to finish
first$ = forkJoin([api1, api2]);  // wait for ALL, get last value of each

🎯 Subjects — pick the right one

  • Subject — no initial value, late subscribers miss past emissions
  • BehaviorSubject(initial) — replays current value to new subscribers (most used for state)
  • ReplaySubject(n) — replays last N values
  • AsyncSubject — emits final value on complete only

🎯 Subscription Cleanup (memory leaks)

// 1. takeUntilDestroyed (Angular 16+) — cleanest
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

constructor() {
  this.api.stream$.pipe(takeUntilDestroyed()).subscribe();
}

// 2. async pipe (let Angular handle subscription)
<div>{{ users$ | async | json }}</div>

// 3. classic takeUntil pattern
private destroy$ = new Subject<void>();
ngOnInit() { this.api.stream$.pipe(takeUntil(this.destroy$)).subscribe(); }
ngOnDestroy() { this.destroy$.next(); this.destroy$.complete(); }

Reactive Forms scale better than Template-Driven — better for complex validation, dynamic forms, async checks. Master FormBuilder + custom validators + CVA.

private fb = inject(FormBuilder);
form = this.fb.group({
  email: ['', [Validators.required, Validators.email]],
  password: ['', [Validators.required, Validators.minLength(8)]],
  addresses: this.fb.array([])

get addresses() { return this.form.get('addresses') as FormArray; }
addAddress() {
  this.addresses.push(this.fb.group({
    line1: ['', Validators.required],
    pincode: ['', [Validators.pattern(/^\d{6}$/)]]
  }));
}

// custom sync validator
function noWhitespace(c: AbstractControl) {
  return (c.value || '').trim().length === 0 ? { whitespace: true } : null;
}

// async validator (e.g., email uniqueness)
export class EmailUniqueValidator {
  constructor(private api: ApiService) {}
  validate = (c: AbstractControl) =>
    this.api.checkEmail(c.value).pipe(map(taken => taken ? { taken: true } : null));
}

6 HttpClient + Typed Services

Wrap HttpClient in typed services that mirror your .NET API. Match DTOs from Phase 3 for type safety end-to-end.

// match .NET DTO
export interface Product {
  id: number;
  name: string;
  price: number;
  createdAt: string;
}

@Injectable({ providedIn: 'root' })
export class ProductService {
  private http = inject(HttpClient);
  private base = `${environment.apiUrl}/api/products`;

  list(query?: { page?: number; size?: number }): Observable<PagedResult<Product>> {
    return this.http.get<PagedResult<Product>>(this.base, { params: query as any });
  }
  get(id: number) { return this.http.get<Product>(`${this.base}/${id}`); }
  create(dto: CreateProductDto) { return this.http.post<Product>(this.base, dto); }
  update(id: number, dto: UpdateProductDto) { return this.http.put<void>(`${this.base}/${id}`, dto); }
  delete(id: number) { return this.http.delete<void>(`${this.base}/${id}`); }
}

Environment Files (mirror appsettings.json)

export const environment = { production: false, apiUrl: 'https://localhost:5001' };

// environments/environment.prod.ts

7 Interceptors — Auth, Error, Loader, Retry

Interceptors are the Angular equivalent of .NET middleware — sit in the HTTP pipeline, transform every request/response.

// modern functional interceptor (Angular 15+)
export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const token = inject(AuthService).token();
  const cloned = token
    ? req.clone({ headers: req.headers.set('Authorization', `Bearer ${token}`) })
    : req;
  return next(cloned);
};

// error interceptor — surface .NET ProblemDetails
export const errorInterceptor: HttpInterceptorFn = (req, next) => {
  const toast = inject(ToastService);
  return next(req).pipe(
    catchError((err: HttpErrorResponse) => {
      if (err.status === 401) inject(Router).navigate(['/login']);
      else if (err.status === 403) toast.error('Forbidden');
      else if (err.error?.title) toast.error(err.error.title);
      else toast.error('Network error');
      return throwError(() => err);
    })
  );
};

// register
provideHttpClient(withInterceptors([authInterceptor, errorInterceptor]))

9 JWT Authentication End-to-End ⭐

Build the complete flow: login → store token → attach to every request → refresh on 401 → guards on routes → role-based UI.

@Injectable({ providedIn: 'root' })
export class AuthService {
  private http = inject(HttpClient);
  private router = inject(Router);

  token = signal<string | null>(localStorage.getItem('token'));
  user = computed(() => this.parseJwt(this.token()));
  isAuthed = computed(() => !!this.user());
  isAdmin = computed(() => this.user()?.role === 'Admin');

  login(email: string, password: string) {
    return this.http.post<{ token: string; refreshToken: string }>(
      `${env.apiUrl}/api/auth/login`, { email, password }
    ).pipe(tap(r => this.setToken(r.token, r.refreshToken)));
  }

  refresh() {
    const rt = localStorage.getItem('rt');
    return this.http.post<{ token: string }>(`${env.apiUrl}/api/auth/refresh`, { rt })
      .pipe(tap(r => this.setToken(r.token)));
  }

  logout() {
    localStorage.removeItem('token'); localStorage.removeItem('rt');
    this.token.set(null);
    this.router.navigate(['/login']);
  }

  private setToken(t: string, rt?: string) {
    localStorage.setItem('token', t);
    if (rt) localStorage.setItem('rt', rt);
    this.token.set(t);
  }

  private parseJwt(token: string | null) {
    if (!token) return null;
    try { return JSON.parse(atob(token.split('.')[1])); }
    catch { return null; }
  }
}

// route guard
export const authGuard: CanActivateFn = () => {
  const auth = inject(AuthService);
  if (auth.isAuthed()) return true;
  inject(Router).navigate(['/login']);
  return false;
};

export const adminGuard: CanActivateFn = () => inject(AuthService).isAdmin();

🎯 Refresh Token Pattern (silent renewal)

  • Backend issues short-lived JWT (15 min) + long-lived refresh token (DB-stored, can revoke)
  • Auth interceptor catches 401 → calls /refresh → retries original request
  • Use switchMap to chain refresh + retry; queue concurrent 401s with a shared refresh observable
⚠️ Gotcha: localStorage tokens are vulnerable to XSS. HttpOnly cookies are safer but need CSRF protection. Pick one and document the trade-off — interviewers ask this.

10 SignalR — Real-Time

SignalR = .NET's WebSocket abstraction. The @microsoft/signalr JS client connects from Angular for chat, notifications, live dashboards.

npm i @microsoft/signalr

// service
import * as signalR from '@microsoft/signalr';

@Injectable({ providedIn: 'root' })
export class NotificationHub {
  private hub: signalR.HubConnection;
  notifications = signal<Notification[]>([]);

  constructor(private auth: AuthService) {
    this.hub = new signalR.HubConnectionBuilder()
      .withUrl(`${env.apiUrl}/hubs/notify`, { accessTokenFactory: () => auth.token()! })
      .withAutomaticReconnect()
      .build();

    this.hub.on('NewNotification', (n: Notification) =>
      this.notifications.update(arr => [n, ...arr]));
  }

  start() { return this.hub.start(); }
  stop()  { return this.hub.stop();  }
}

Backend setup (preview): app.MapHub<NotificationHub>("/hubs/notify"); + builder.Services.AddSignalR();

11 NgRx + Signal Store

For complex multi-component state. Choose based on team and Angular version.

  • NgRx Store — Redux pattern (Actions, Reducers, Effects, Selectors). Powerful but boilerplate-heavy.
  • NgRx Signal Store — modern, leverages Angular signals. Less boilerplate, growing fast.
  • Plain services + Signals/BehaviorSubject — for small to medium apps
// NgRx Signal Store (Angular 17+)
export const CartStore = signalStore(
  withState({ items: [] as CartItem[], loading: false }),
  withComputed(({ items }) => ({
    total: computed(() => items().reduce((s, i) => s + i.price, 0)),
    count: computed(() => items().length)
  })),
  withMethods((store, api = inject(CartApi)) => ({
    async load() {
      patchState(store, { loading: true });
      const items = await firstValueFrom(api.getCart());
      patchState(store, { items, loading: false });
    }
  }))
);

🎤 Interview Q&A — Phase 5

Q. What are Angular Signals and how they differ from RxJS?

Definition:

  • Signals = new reactive primitive in Angular 16+
  • Function that hold a value
  • Read with count(), write with count.set(5) or count.update(c => c + 1)

How it works:

  • Angular track where each signal is read
  • When value change — only those components re-evaluate
  • More efficient than old zone.js change detection

Signals vs Observables:

SignalObservable
SynchronousAsynchronous
Always has valueNo initial value
Eager (auto running)Lazy (need subscribe)
For stateFor event streams

When to use:

  • Signals — component state, derived values
  • RxJS — HTTP, WebSocket, timers, mouse events

Related:

  • computed() — derived signals
  • effect() — run side-effects when signal change

Note: Signals are foundation of "zoneless" Angular — future direction.

Q. Explain switchMap vs mergeMap vs concatMap vs exhaustMap?

Definition:

  • 4 "higher-order mapping" RxJS operators
  • Flatten Observable of Observables
  • Differ in how they handle overlap

Compare Table:

OperatorBehaviorUse For
switchMapCancel previous, use newType-ahead search
mergeMapRun all in parallelIndependent calls (analytics)
concatMapQueue, run one by one in orderSave edits in order
exhaustMapIgnore new while runningLogin button (no double click)

Quick interview trick:

  • Q: "Search box that fires HTTP — which operator?"
  • A: switchMap (cancel old searches)
Q. What is OnPush change detection and when to use?

Default vs OnPush:

  • Default — Angular check every component on every async event (slow in big apps)
  • OnPush — only re-check when something concrete change

OnPush Triggers (5 things):

  1. @Input() reference change (new object, not mutation)
  2. Event handler in this component fire
  3. async pipe in template emit
  4. Signal read in template update
  5. Manual cdr.markForCheck()

Performance Win:

  • Instead of checking 500 components on every click
  • Only check the few that actually changed

BIG Catch — Immutability:

  • arr.push(x) — OnPush NOT notice (same reference)
  • arr = [...arr, x] — OnPush detect (new reference)
  • Or use Signals

Note: Use OnPush as default on all "leaf" components, especially in lists with *ngFor.

Q. How to implement JWT auth in Angular?

5 Step Flow:

  1. AuthService — hold token as Signal, persist to localStorage on login
  2. HTTP Interceptor — add Authorization: Bearer <token> to every request
  3. Error Interceptor — catch 401, call refresh endpoint, retry original
  4. CanActivate Guards — protect routes, check isAuthed()
  5. Logout — clear localStorage, navigate to /login

AuthService Properties:

  • Computed: isAuthed(), isAdmin()
  • Login method post credentials, get { token, refreshToken }
  • Store both tokens

Security Notes:

  • localStorage = vulnerable to XSS — sanitize user input
  • HttpOnly cookies = no XSS risk but CSRF risk
  • Decode JWT on client ONLY for UI hints (show user name)
  • Real security check happen on server

Java parallel:

  • Same JWT pattern as Java backend
  • Token in header, validated on server
Q. How do interceptors compare to .NET middleware?

Same Concept:

  • Both are pipelines of handlers
  • Process every HTTP request and response
  • Cross-cut concerns — auth, logging, error, retry
  • Without touching every call site

Angular Interceptor:

  • Function or class
  • Register with provideHttpClient(withInterceptors([...]))
  • Receive request, clone and modify
  • Call next(req) to pass down chain
  • Handle errors with catchError

.NET Middleware:

  • Register in Program.cs with app.Use...()
  • Receive HttpContext
  • Do work, call next()

Both:

  • Ordered pipeline
  • Can short-circuit (skip rest by not calling next)
  • Can modify response on way back

Mental model:

  • Interceptor = "client-side middleware" (wrap OUT requests)
  • .NET Middleware = wrap IN requests
  • Together = full pipeline from Angular → server → back
Q. What is RxJS memory leak and how to prevent?

Why it happens:

  • Call .subscribe() and never unsubscribe
  • Observable hold reference to callback
  • Callback hold this (your component)
  • Component never garbage collected after destroy

3 Modern Ways to Prevent:

1. takeUntilDestroyed (Angular 16+) — Cleanest:

this.api.stream$.pipe(takeUntilDestroyed()).subscribe();
  • Pipe inside constructor
  • Auto-complete when component destroyed

2. async pipe in template — Best for Display:

{{ users$ | async }}
  • Angular subscribe when template render
  • Auto unsubscribe when removed

3. takeUntil Pattern (Classic):

private destroy$ = new Subject<void>();
this.api.stream$.pipe(takeUntil(this.destroy$)).subscribe();
ngOnDestroy() { this.destroy$.next(); this.destroy$.complete(); }

Avoid: Storing subscription objects manually — easy to forget.

Q. Standalone components vs NgModules?

Definition:

  • Standalone — modern way (Angular 14+, default in Angular 17)
  • NgModules — old system

Old NgModule System:

  • Declare every component, directive, pipe in module first
  • Lead to fat NgModules and boilerplate

Standalone System:

  • Component declare own dependencies in imports array
  • No module file needed
  • Example: ProductCard import CommonModule and RouterLink directly

Benefits:

  • Smaller bundles (better tree-shaking)
  • Faster app startup
  • Simpler mental model
  • Easier lazy loading via loadComponent

New Bootstrap:

bootstrapApplication(AppComponent, { providers: [...] })

Note: Can use NgModules in legacy code — they coexist. For new code, default to standalone.

Q. How to handle CORS during development?

Why it happens:

  • Angular on localhost:4200
  • .NET API on localhost:5001
  • Different ports = different origin = CORS error

Proper Fix — Server Side:

  • In .NET API: builder.Services.AddCors
  • Allow Angular origin (http://localhost:4200)
  • Add app.UseCors("PolicyName") before UseAuthentication
  • Specify origins, methods, headers explicitly
  • Don't use AllowAnyOrigin in production with credentials

Dev-Only Shortcut — Angular Proxy:

  • Create proxy.conf.json
  • Point /api to https://localhost:5001
  • Run ng serve --proxy-config proxy.conf.json
  • Angular dev server forward calls to .NET
  • Browser see same origin — no CORS

Note: Proxy work in dev only. Still need server CORS for production.

Q. Reactive Forms vs Template-Driven?

Definition: Angular has 2 ways to build forms.

Template-Driven Forms:

  • Use directives in HTML template — ngModel, required
  • Angular build FormGroup behind scene
  • Easy for simple forms (contact us)

Reactive Forms (Better):

  • Defined in component class using FormBuilder
  • Explicitly create FormGroup, FormControl, FormArray
  • Template bind with [formGroup] and formControlName

Why Reactive Better:

  • Stronger typing
  • Easier dynamic forms (add/remove fields)
  • Built-in async validators (check email unique with server)
  • Simpler unit testing (no template render needed)
  • Access to RxJS observables for value/status changes

Rule: Don't mix both styles in same form — pick one and stick.

Note: Most production codebases use Reactive Forms exclusively.

⚠️ Common Mistakes

  • Forgetting to unsubscribe → memory leaks (use async pipe or takeUntilDestroyed)
  • Mutating arrays in OnPush components — change detection misses it
  • Calling subscribe inside subscribe instead of switchMap
  • Storing JWT in localStorage without considering XSS (or in cookies without CSRF)
  • Not using trackBy in *ngFor → DOM thrash on every render
  • Mixing Reactive and Template-driven forms in same project
  • Ignoring strict mode in tsconfig (you should always be strict)
  • Using mergeMap when you wanted switchMap (race conditions in search)
