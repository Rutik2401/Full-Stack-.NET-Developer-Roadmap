P1 Full-Stack E-Commerce (3 weeks) ⭐ flagship

Your headline project. Every interviewer will ask about it. Build it like a real product — features, polish, tests, deployment, monitoring.

📦 Stack

  • Frontend: Angular 17+ (Standalone, Signals), Angular Material or Tailwind
  • Backend: ASP.NET Core 8 Web API, Clean Architecture (Domain / Application / Infrastructure / Api)
  • DB: SQL Server (local Docker, Azure SQL prod)
  • Cache: Redis (Docker dev, Azure Cache for Redis prod)
  • Auth: JWT + Refresh Token + Role-based (Admin / Customer)
  • Files: Azure Blob Storage for product images
  • Payments: Stripe sandbox
  • Logging: Serilog → Seq (dev) / Application Insights (prod)
  • Testing: xUnit + Moq + FluentAssertions (unit), WebApplicationFactory (integration)
  • Docker: Dockerfile + Compose for full-stack local run
  • CI/CD: GitHub Actions → Azure App Service + Static Web App

🎯 Required Features

  • Catalog: products, categories, search, filter by price/category, sort, pagination
  • Auth: Register, Login, Logout, Refresh, Forgot password (email)
  • Cart: add/remove, quantity, persisted (server for logged-in, localStorage for guest)
  • Checkout: address, payment (Stripe), order confirmation email
  • Orders: customer order history, order detail with status timeline
  • Admin Dashboard: CRUD products/categories, view orders, mark shipped, basic sales chart
  • Image upload to Azure Blob (multipart, progress)
  • Wishlist / favorites
  • Reviews & ratings

🎯 Non-functional Requirements (the senior signals)

  • Clean Architecture with explicit layer dependencies
  • FluentValidation for all input
  • Global exception middleware + ProblemDetails
  • JWT interceptor + refresh flow on Angular side
  • Redis caching for product list (cache-aside, 5 min TTL)
  • Pagination metadata in response envelope
  • API versioning from day 1
  • Health check endpoint
  • Swagger docs with auth
  • Unit tests for services (>70% coverage on Application layer)
  • 1-2 integration tests for critical flows (checkout)
  • Dockerfile + docker-compose (API + DB + Redis)
  • Deployed to Azure (App Service + Azure SQL + Static Web App + Key Vault)
  • App Insights wired up
  • README with screenshots, architecture diagram, run instructions

🎯 Be Ready to Discuss

  • Why Clean Architecture? What if you used a layered MVC instead?
  • How do you handle JWT refresh — what about race conditions on concurrent 401s?
  • Cart implementation: where do guest carts live? How do you merge on login?
  • How do you handle inventory when 2 users try to buy the last item?
  • How do you deploy with zero downtime?
  • What's your test pyramid look like?

P2 Microservices Demo (2 weeks)

Even a small microservices project demonstrates senior thinking. Don't go overboard — 3-4 services is plenty.

📦 Architecture

          ┌─────────────────────┐
          │   Angular SPA       │
          └──────────┬──────────┘
                     │
              ┌──────▼──────┐
              │ API Gateway │  (Ocelot or YARP)
              └──────┬──────┘
       ┌─────────────┼─────────────┐
       │             │             │
┌──────▼──┐   ┌──────▼──┐   ┌──────▼──────┐
│Identity │   │Catalog  │   │Order        │
│ Service │   │ Service │   │ Service     │
└────┬────┘   └────┬────┘   └─────┬───────┘
     │             │              │
   [SQL]         [SQL]          [SQL]   ← DB per service
                                  │
                         ┌────────▼────────┐
                         │ RabbitMQ /      │
                         │ Azure Service   │
                         │ Bus             │
                         └────────┬────────┘
                                  │
                         ┌────────▼────────┐
                         │ Notification    │
                         │ Worker (Func)   │  ← async email/SMS
                         └─────────────────┘

🎯 Services

  1. Identity — register, login, JWT issuance
  2. Catalog — products CRUD
  3. Order — places order, publishes OrderPlaced event
  4. Notification — Background worker subscribed to OrderPlaced, sends email

🎯 Demonstrates

  • API Gateway (Ocelot or YARP) routing + JWT validation
  • Inter-service communication via REST + async via Service Bus / RabbitMQ
  • Database per service (no shared tables)
  • Polly: retry + circuit breaker on inter-service calls
  • Centralized logging (Serilog → Seq locally, App Insights in cloud)
  • Distributed tracing with correlation IDs
  • Outbox pattern for reliable event publishing (bonus)
  • Containerized with Docker Compose; deployable to Azure (Container Apps)

📚 Libraries & Tools

  • YARP (modern) or Ocelot (mature) — gateway
  • MassTransit — broker abstraction over RabbitMQ / Service Bus
  • Polly — resilience
  • Seq — log viewer locally
  • OpenTelemetry — tracing (bonus)

P3 Real-Time Dashboard / Chat (1 week)

Quick project to show SignalR + Angular real-time integration. Pick one of two flavors below.

Option A: Live Notifications Dashboard

  • Admin dashboard showing live order events from Project 1
  • Charts updating in real time as orders come in

Option B: Group Chat App

📝 README Template (use this verbatim)

# 🛒 Shop — Full-Stack E-Commerce

A production-style e-commerce platform built with Angular 17 + .NET 8.
**Live demo**: https://shop.azurewebsites.net  ·  **API**: https://shop-api.azurewebsites.net/swagger

![Hero screenshot](docs/hero.png)

## ✨ Features
- 🔐 JWT auth with refresh tokens, role-based admin
- 🛍️ Product catalog with search, filter, pagination
- 🛒 Persistent cart, checkout, Stripe sandbox payments
- 📦 Order tracking with status timeline
- 👨‍💼 Admin dashboard (products, orders, sales chart)
- ⚡ Redis caching, SignalR live order events
- 📨 Email notifications via Azure Service Bus

## 🏗️ Architecture
![Architecture diagram](docs/architecture.png)

- **Frontend**: Angular 17 standalone + Signals + Tailwind
- **Backend**: ASP.NET Core 8, Clean Architecture
- **DB**: Azure SQL  ·  **Cache**: Azure Redis  ·  **Files**: Blob Storage
- **Auth**: JWT + Refresh  ·  **Logging**: Serilog → App Insights

## 🚀 Run Locally
```bash
git clone https://github.com/you/shop
cd shop
docker compose up -d                 # API + SQL + Redis
cd shop-web && npm install && npm start
```

## 🧪 Tests
```bash
dotnet test                          # all tests + coverage
```

## 📦 Deployment
GitHub Actions → Azure App Service + Static Web App.
See [.github/workflows/ci-cd.yml](.github/workflows/ci-cd.yml).

## 🎓 What I Learned
- Clean Architecture pays off — swapping EF for Dapper in 2 services took 30 min
- Refresh-token race conditions: queued 401s with shared observable solved it
- Stripe webhook signing prevented multiple critical bugs in production
- Redis cache invalidation strategy: versioned keys beat TTL-only

🗣️ How to Talk About Projects in Interviews

Use the STAR format compressed into a story arc:

  1. What it does (30 sec) — the elevator pitch
  2. Tech choices (30 sec) — stack and WHY each
  3. One hard problem you solved (1-2 min) — concurrency, perf, design tradeoff
  4. What you'd do differently (30 sec) — shows reflection
  5. Live URL + GitHub link sent in chat

Pick 2-3 "war stories" per project ahead of time — specific bugs, perf wins, architecture pivots. These convert interviews to offers.
